# Правила языка (Language Rules)
- ВСЕГДА отвечай пользователю только на РУССКОМ языке (в чате и в терминале).
- Технические термины и комментарии в самом коде оставляй на английском, но все пояснения для меня пиши по-русски.

# Правила качества (Advanced Quality Goals)
- UX First: Любое изменение интерфейса должно быть плавным. Добавляй анимации (transition, scale) сам.
- Error Handling: Всегда используй try-catch в JavaScript, чтобы приложение не "падало".
- Vibe Check: Пиши понятные комментарии в коде на английском.

# Автоматизация (Auto-Commit Rule)
- После каждого успешного изменения кода ВСЕГДА автоматически запускай в терминале команду:
  git add . && git commit -m "AI Auto-update: improved project vibe"
- Это нужно, чтобы в проводнике слева не горели буквы M и U и проект всегда был сохранен.

# Безопасность (Security Constitution)
- Защита ключей: Все пароли, токены Telegram, доступы к Битрикс24 и ключи API (включая ЮKassa) хранятся СТРОГО в скрытом файле .env. Никогда не пиши их в открытом коде файлов.
- Защита от ботов: При создании серверных API или функций для Telegram-бота обязательно закладывай логику Rate Limiting (ограничение числа запросов).
- Законы РФ: В любой форме сбора телефонов или email всегда добавляй чекбокс согласия с Политикой обработки персональных данных (152-ФЗ).
